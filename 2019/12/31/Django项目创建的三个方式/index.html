Django项目创建的三个方式

2019-12-31

安装Django：

pip install django==2.1.14 -i https://pypi.doubanio.com/simple/

第一种

1) 创建一个项目django01(打开终端)

django-admin startproject django01

2) 使用PyCharm打开项目并创建虚拟环境(依次输入以下)

Terminal --> python -m venv venv --> "venv install django==2.1.14"

3) 运行项目(两种运行方式)

~ python manage.py runserver
~ Add Configuration --> + --> Python --> Script Path manage.py --> Parametes(runserver)

第二种

1) 用PyCharm创建一个普通python项目
2) 创建一个虚拟环境
3) 在虚拟环境中安装Django的依赖项

pip install django==2.1.14

3) 把Python项目变为Django项目(千万别忘了最后的点)

django-admin startproject django02 .

4) 运行项目

~ python manage.py runserver
~ Add Configuration --> + --> Python --> Script Path manage.py --> Parametes(runserver)

第三种

1) 克隆项目到本地(两种方式)

~ 使用PyCharm的"get from version control"
~ git clone git@gitee.com:yydafx/django02.git

2) 创建虚拟环境
3) 重建依赖项

pip install -r requirements.txt
补充(对本地代码实施版本控制病史上传到版本控制器)
准备工作：先创建一个Django项目
    在Django根目录下创建两个文件：
        a、依赖项文件(requirements.txt)
            执行 pip freeze > requirements.txt

        b、忽略文件(.gitignore)
进入到django项目的根目录下
    git init  -->    将普通文件变成版本仓库
    git add .    -->    将文件从工作区同步到暂存区
    git commint -m '提交信息'    -->    将文件提交到本地仓库
    git remote add origin git@gitee.com:yydafx/django02.git    -->    绑定远端仓库
    git push -u origin master    -->    将本地代码推到服务器

    git status    -->    查看...
    git log    -->    查看日志
